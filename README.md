# Homework

## 1. Capacity Test

Questo homework studia la **capacità di un web server** quando viene sottoposto a un carico crescente di richieste.

Attraverso strumenti di **load generation** e monitoraggio del sistema vengono analizzate alcune metriche fondamentali come:

- **Throughput**
- **Response Time**
- **Power**

L’obiettivo è individuare i punti operativi principali del sistema, come **Knee Capacity** e **Usable Capacity**, e valutare il comportamento del server sotto carico concorrente tramite il **Fairness Index**.

---

## 2. PCA e Clustering

In questo homework vengono utilizzate tecniche di **analisi dei dati** per studiare il comportamento del sistema tramite le metriche raccolte dal sistema operativo.

L’analisi prevede:

- riduzione della dimensionalità tramite **Principal Component Analysis (PCA)**
- individuazione di gruppi di comportamento simili tramite **clustering gerarchico**

Queste tecniche permettono di individuare strutture nei dati e di generare **workload sintetici rappresentativi**.

---

## 3. Workload Characterization

Questo homework si concentra sulla **caratterizzazione di un workload reale** generato verso un sistema.

Il processo prevede tre fasi principali:

- analisi del **workload reale**
- generazione di un **workload sintetico**
- **validazione statistica** per verificare che il workload sintetico rappresenti correttamente quello originale

---

## 4. Regression Analysis

Questo homework analizza l’evoluzione nel tempo di alcune **metriche di sistema** tramite tecniche di **analisi statistica**.

Vengono utilizzati **modelli di regressione** e **test statistici** per individuare eventuali **trend temporali** nei dati osservati.

---

## 5. Design of Experiments (DoE)

In questo homework viene applicata la metodologia **Design of Experiments (DoE)** per studiare l’influenza di diversi parametri sulle prestazioni del sistema.

L’obiettivo è identificare **quali fattori influenzano maggiormente le performance** e valutarne la **significatività statistica**.

---

## 6. Benchmark

Questo homework utilizza un **benchmark computazionale** per confrontare le prestazioni del sistema in diversi scenari.

L’analisi include metodi di **statistica descrittiva** e tecniche di confronto tra dataset per valutare eventuali differenze di prestazioni.

---

## 7. Reliability Block Diagram e Fault Tree

In questo homework vengono utilizzati modelli di **affidabilità dei sistemi** per analizzare la probabilità di guasto di un sistema composto da più componenti.

L’analisi viene svolta tramite:

- **Reliability Block Diagrams (RBD)**
- **Fault Tree Analysis (FTA)**

Questi strumenti permettono di studiare come i guasti dei singoli componenti influenzano il comportamento dell’intero sistema.

---

## 8. Failure Data Analysis (FFDA)

L’ultimo homework analizza dataset di **failure reali** provenienti da sistemi di calcolo ad alte prestazioni.

L’obiettivo è studiare la **distribuzione dei guasti** e comprendere il comportamento dei sistemi su larga scala tramite tecniche di **failure data analysis**.
